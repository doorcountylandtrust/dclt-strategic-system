# CONTENT MIGRATION & REWRITE PLAN (Sept–Oct 2025)

This plan outlines the steps needed to review, assess, rewrite, and structure all content for the new DCLT website.

---

## OVERVIEW

- **Goal:** Rebuild all core pages with intentional content strategy, modernized messaging, and mapped user journeys
- **Source Inputs:** Markdown exports, screenshot archive, XML recovery, stakeholder interviews, past campaigns
- **Outputs:** Modular, rewritten, and approved content blocks mapped to wireframes and IA

---

## PHASE 1 – Content Inventory & Audit (Week 1–2)

| Task | Owner | Notes |
|------|-------|-------|
| Organize all recovered markdown files | Don | Pages vs Posts, rename if needed |
| Organize screenshot archive and confirm hierarchy | Don | Visual context, screenshots used for missing content |
| Build Master Content Audit Table | Don | Columns: Title, Status, Keep, Rework, CTA, Page Type, Screenshot Ref |
| Tag content by primary audience + funnel stage | Don | Reference Personas |
| Identify content gaps | Don | What new pages/modules are missing? |
| Review past analytics if available | Don | Flag underperforming pages |

---

## PHASE 2 – Rewrite, Trim, Improve (Week 2–4)

| Task | Owner | Notes |
|------|-------|-------|
| Set content voice/tone guidance | Don | Align with brand guidelines |
| Rewrite top-level nav pages first | Don | Homepage, About, Preserves, Support Us |
| Rewrite key subpages using new modular content blocks | Don | Use Hero, CTA, Story modules, etc. |
| Trim or remove outdated content | Don | Archive strategy for legacy pages |
| Write new strategic content (e.g., Find Your Path, Giving Tree) | Don | Use planned new sections |
| Align rewritten content to wireframes | Don | Content length, layout, flow |
| Internal reviews of rewritten pages | Staff | Assign by section or department |
| Revise based on staff review | Don | Build feedback into clean v2 drafts |

---

## PHASE 3 – Final Migration Prep (Week 5–6)

| Task | Owner | Notes |
|------|-------|-------|
| Match final content to modular field structure | Don | Per block types: CTA, Hero, FAQ, etc. |
| Place rewritten content into migration-ready format | Don | Markdown, ACF import JSON, or CMS input |
| Internal QA testing of placeholder content on dev site | Don + Staff | Layout + editorial bugs |
| Flag missing assets (images, PDFs) for upload | Don | Create checklist |
| Prep fallback copy for unfinished sections | Don | Basic versions ready by soft launch |

---

## GOAL: ALL FINAL CONTENT READY BY JANUARY 2026
